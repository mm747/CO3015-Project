Use MySQL workbench to access the Department MySQL server.

Configuration

To configure the connection to MySQL workbench, create a new connection and configure it as indicated below depending on whether you are working from a machine in the lab or off-campus (including eduroam).

From campus (lab)

Replace the following variables in the class DbConfig.java as follows:

  • HOSTNAME with mysql.mcscw3.le.ac.uk
  • PORT with 3306
  • USERNAME with your username
  • PASSWORD with your MySQL password, stored in .my.cnf file (open a terminal console and run the command cat .my.cnf to view it).
Off-campus (including eduroam)

From a terminal console: run the command ssh -fNg -L 3307:mysql.mcscw3.le.ac.uk:3306 ${USERNAME}@xanthus.mcscw3.le.ac.uk, where ${USERNAME} has to be replaced with your actual user name. This command creates a SSH tunnel from a port (3307 in our command) on the local machine to the MySQL server port (3306 in our command) on the MySQL host machine. The server xanthus.mcscw3.le.ac.uk is used as a gateway machine to reach the database server mysql.mcscw3.le.ac.uk. The command will ask for your Linux password. If the tunnel is created successfully, you should be able to use your terminal console as usual and no success message is going to be shown. This configuration is documented in more detail here.

Replace the following variables in the DbConfig.java as follows:

  • HOSTNAME with 127.0.0.1
  • PORT with 3307
  • USERNAME as above.
