Description

In this sprint 4, we are going to combine the different tools and techniques that have been discussed in sprints 1, 2 and 3 by:

  • using Gradle to automate the configuration of our project [sprint 1];
  • combining the miniproject that was developed in sprint 2 and the one that was tested in sprint 3 in order to equip our online shop with a sign-up feature [sprints 2 and 3];
  • using Groovy to specify tests [sprint 3];
  • using test-driven development and behaviour driven development based on Spock to drive the development a new feature [sprint 3].

In addition, we will a data layer and a security layer to the resulting web app so that the data on the online shop is stored in a MySQL database and we can use the web app securely. The development of these features will be driven by feature methods specifying BDD scenarios, which will be used as acceptance criteria of how well the miniproject has been implemented. That is, the BDD scenarios are the marking criteria.

The miniproject consists of two parts:

  • checkpoint: generation of database and creation of data in the database with Spring Data;
  • release:
    • security: secure connection, authentication and authorization;
    • miniproject: integration of a data layer in the work done in the web app resulting from sprints 2, 3 and 4.

The solution of the checkpoint of sprint 4 will be used for building the app, where we will focus on integration. Hence, the codebase for the miniproject will be released after the deadline of the checkpoint. This will facilitate the participation in the second part of the worksheet even if there were errors in the configuration of the data layer, in the checkpoint submission.

⭐️ Spring Security [30 marks]

Solve this exercise in project miniproject.security.

Secure connection [5 marks]

  • Create a new digital certificate using keytool with your University username as the full name in the certificate.
  • Configure the web app in order to use it via a secure connection only, using the protocol https. The files that need to be configured are:
    • src/main/java/eMarket/SecurityConfig.java
    • src/main/resources/application.properties

Authentication [15 marks]

Configure the authentication in the app using information from LoginController. In the file src/main/java/eMarket/SecurityConfig.java, configure:

  • login()
    • login must be processed by /login
    • the request to show the view for entering the credentials is /login (implemented by the Spring framework)
    • when login is successful, the user must be redirected to /success-login
    • when authentication fails, users must be redirected to /login-form
  • logout(), when login out,
    • the information about the HTTP session must be deleted;
    • the user must be redirected to /login-form

Desk check that the user can log in and log out. Checking that if we try to access /system/ without being authenticated, we are redirected to /login-form.

The web app must work as in sprint 2 but instead of retrieving data from an abstraction of a data store - the static object EMarketApp.getStore(), - the data must be stored and retrieved from your database via a Spring Data repository.

Solve this exercise in project miniproject. Proceed as follows:
